Overview of the article

It is well known that Laravel is coming with built-in Vue JS to support its front end besides Blade. We'll using both the SPA power of Vue JS and API power of Laravel. In this tutorial, you will make this application in less than one hour.

Requirements
  • Composer, a dependency manager for PHP.
  • Node JS, we'll gonna use it's NPM (Node Package Manager) to install the Javascript dependencies.
  • Laravel, as we gonna use this alongside with Vue JS to create the application.
Step 1: Installing the Laravel and Vue UI scaffolding

At this point, I'm assuming you have already installed Composer and Node JS already. If you really haven't please check their documentation about how to install the individual application. Let's install the Laravel Installer so you could install the Laravel in the other time.

composer global require laravel/installer

The Composer then will install the Laravel to the Composer's directory. The directory is different based on your operating system. If you're using Windows, the Composer directory will automatically added into your System Environment path. In my case which I'm using Linux, I have to manually set my bash path to the Composer directory.

Let's put the Laravel installation aside, we need to make new Laravel project.

laravel new spa-todo-laravue && cd spa-todo-laravue

Command above create a new Laravel project called "spa-todo-laravue" then prompt the terminal to go to the project directory. Next one is installing the scaffolding. The scaffolding is used to install the Vue dependencies within Laravel. Run command below to install the scaffolding.

composer require laravel/ui

After the installation is done, then generate the basic scaffolding using the command below. The scaffolding will be placed on /resources/js/.

php artisan ui vue

The installation is basically done. But we actually need two other JavaScript dependencies to run Vue JS as a Single Page Application. Both of them are vue-router and axios. They are needed to make the project run as an SPA and to send HTTP Request respectively. Run the command below.

npm install npm install vue-router axios --save-dev npm run dev

The command above is translated as such:

  1. We install all of the dependencies needed for Laravel and Vue scaffolding to the project. This will create huge ammount of library on your /node_modules/ folder. But don't worry, we won't be interacting with those files directly, instead we will use them.
  2. We install another two required dependencies as I mentioned above before the command block, both are needed to run the application as an SPA and to send the HTTP Request respectively. If you are familiar with AJAX, that is technically what axios does. As my friend Ikramullah said, the flag --save-dev is used to save the dependencies to the package.json just in case you are working as a team.
  3. As I cited from Scotch npm run dev - Bootstraps our app, opens it in the browser, reloads the browser whenever we make changes in source.

Make sure you run ALL of the commands above or else, it won't run perfectly.

Step 3: Make a Laravel To-do Model, Controller, and Migration.

If you are new to Laravel, then you might know those components. Model is a file where you will be able to communicate between the application and the database, the Model is a main bridge between them easier. Controller is a file where you could interact which data will be printed out from the database, it basically controls how you want the app to be. Migration is a file where you make up a database table schema, if you're comparing doing this on traditional way using PHPMyAdmin you might feel awkward at the beginning but trust me it is a powerful file.

Below here is how you can create all of those files

// One by one php artisan make:model Todo php artisan make:controller TodoController --resource --model=Todo php artisan make:migration create_todos_table // All at once php artisan make:model Todo -mrc

The command basically translated like this:

  1. Make a model named "Todo"
  2. Make a controller named "TodoController", create it as a resource controller, using model Todo as it's bridge.
  3. Make a migration to create 'todos' table. As a default, table name should be on plural term. It means that 'index' are going to be 'indices', 'user' are going to be 'users', 'product' are going to be 'products' ect.
  4. You can simplify all of the commands above by using the 4th command. But as a note this command need -mrc flag which are translated to: Create a model named "Todo" with a (m)igration, and a (r)esource (c)ontroller.
